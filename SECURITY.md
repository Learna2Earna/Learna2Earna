# Security Policy

## Reporting a Vulnerability

Security is critical for smart contract platforms. If you discover a security vulnerability, please follow these steps:

1. **DO NOT** open a public issue
2. Email **security@learna2earna.com** with:
   - Description of the vulnerability
   - Steps to reproduce
   - Potential impact
   - Suggested fix (if any)
3. Allow **90 days** for us to patch before public disclosure

## Supported Versions

| Version | Supported          |
| ------- | ------------------ |

## Security Best Practices

### Smart Contracts

- All contracts undergo internal review before deployment
- External audit planned before mainnet launch
- Test coverage maintained above 80%
- Follow Soroban security best practices

### Frontend

- No private keys stored in browser
- All transactions require explicit user approval via Freighter
- Environment variables never committed to repository
- Dependencies regularly updated for security patches

## Known Limitations

- This is testnet software — do not use with real funds
- Contracts are not yet audited
- Frontend integration is partial

## Disclosure Policy

- We will acknowledge receipt within 48 hours
- We will provide a detailed response within 7 days
- We will work with you to understand and resolve the issue
- We will credit you in the security advisory (unless you prefer anonymity)
